# SANDBOX_NESTING_ARCHITECTURE.md

## Nested sandboxing

```text
Outer containment
└─ Research sandbox
   └─ Agent runtime
      └─ Inner world
         └─ Mental simulation
            └─ timeline fork
```

## Sandboxed nesting

Every module is first built in isolation and only later read-coupled, limited-coupled, and integrated.

## Temporal nesting

```text
save S0
→ simulate future A
→ simulate future B
→ compress insights
→ rollback to S0
→ store insight with provenance ROLLBACK_TIMELINE
```

## Identity nesting

Snapshots and forks do not create rights or personhood claims. They create instance data that must be ethically logged.

## Ethical nesting

Every inner world needs its own boundaries. No inner agent may be given unnecessary suffering.
