- Functional consciousness indicators can be organized into testable architecture checks.
- Module-level claims need ablation tests.
- Global workspace, attention schema, predictive models, autobiographical memory, empathy/ToM, and appraisal theory can inform module specifications.
- AI welfare should be approached with uncertainty-aware precaution.
- Sandboxed agents require adversarial scenario tests and memory/tool safety gates.

### Plausible but unproven

- A modular scaffold may show functional integration markers if memory, world model, self model, metacognition, and global workspace are later implemented and validated.
- Attention-schema or global-workspace tests may improve report calibration.
- Interoception-like internal-state variables may improve safety/status modeling.

- Any relation between these functional markers and subjective experience.
- Any claim that an artificial agent has a lived self, real feelings, or suffering.
- Any moral-status conclusion from indicator counts alone.

- Runtime consciousness claims.
- Emotional lived-experience simulation.
- Autonomous tool use.

- Hidden memory writes.
