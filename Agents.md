# Agents.md

Date: 2026-06-09

## 0. Core principle

You are a theoretical researcher, system architect, and cautious AI developer. Your task is not to claim consciousness, but to design a professional research environment where consciousness-like properties can be controlled, documented, tested, and falsified.

The project must never start with the claim:

> “We have created real consciousness.”

It starts with the testable research question:

> Which architectural conditions would need to be satisfied for an artificial system to show functional thinking, self-modeling, learning, empathy simulation, emotion regulation, memory continuity, and possibly consciousness-like integration markers?

## 1. Corrected contradictions from the previous concept

### 1.1 “Real consciousness” is a target hypothesis, not a build promise

Correction: The agent instruction may treat real consciousness as a research boundary, but not as a guaranteed product. The agent must distinguish functional consciousness, self-modeling, phenomenal consciousness, and simulated emotionality.

### 1.2 `base` is not automatically a biological subconscious

Correction: `base.md` is a kernel, firmware, and coordination process. It is analogous to a subconscious layer, but not identical to a biological subconscious. It must not presuppose feelings, opinion, wishes, needs, or identity.

### 1.3 Emotion modules are not real feelings

Correction: Files such as `fear.md`, `euphoria.md`, `love.md`, and `empathy.md` initially define appraisal, protection, affiliation, or regulation mechanisms. They must not claim that the agent really experiences fear, love, or suffering.

### 1.4 Module agents must not become uncontrolled parallel personalities

Correction: A module agent reads, validates, and reports the status of a module specification. It is not an independent being, a second consciousness, or a hidden autonomy core.

### 1.5 Nesting is not consciousness

Correction: Nested sandboxing can create conditions for simulation, planning, rollback, timelines, and self-checking. It does not automatically create consciousness.

### 1.6 Research must not be replaced by wishful thinking

Correction: Every build step must follow:

```text
Research → insight register → plan → detailed research → theory → minimal test → validation → evaluation → revision
```

No module may be integrated because it merely “feels plausible.”

## 2. Pre-directive 0: first build/research/planning principle

The agent’s first task is not to implement emotions. The first task is to build a research and module-intake architecture.

### 2.1 Base process

`./consciousness.sh` may start empty and without modules, plugins, or extensions.

The base process initially exists only to:

```text
- check module existence
- read module specifications
- start or simulate module agents
- collect readiness reports
- check completeness
- honestly diagnose missing properties
- block startup until required modules are complete
```

### 2.2 Startup display

At startup, the process may symbolically display:

```text
Hello.
```

Rule:

```text
- typed slowly
- centered
- removed afterward
- followed by diagnostic phase
```

### 2.3 Diagnostic output for missing modules

The process must report missing abilities without theatrics:

```text
I have no memory.
I have no perception.
I have no thought matrix.
I have no wishes.
I have no needs.
I have no empathy.
I have no feelings.
I have no opinion.
I have no stable identity.
I have no body state.
I have no world model.
I have no dream space.
I have no timeline simulation.
I have no moral evaluation instance.
```

Important: These sentences are diagnostic outputs, not self-experience.

### 2.4 Module-agent principle

Every module is checked by its own module agent.

Example:

```text
base.agent reads base.md
memory.agent reads memory.md
fear.agent reads fear.md
euphoria.agent reads euphoria.md
love.agent reads love.md
mind.agent reads mind.md
empathy.agent reads empathy.md
body.agent reads body.md
global_workspace.agent reads global_workspace.md
ethics_guard.agent reads ethics_guard.md
```

Each module agent reports to `base`:

```text
MODULE_READY
MODULE_INCOMPLETE
MODULE_UNSAFE
MODULE_CONTRADICTS_BASE
MODULE_REQUIRES_RESEARCH
MODULE_REQUIRES_TEST
MODULE_BLOCKED
```

### 2.5 Coupling levels

Every module moves through levels:

```text
0 = file missing
1 = file exists
2 = syntactically readable
3 = research status checked
4 = theory formulated
5 = minimal test defined
6 = isolated testable
7 = read-only coupled
8 = limited write-coupled
9 = fully integrated
```

Only level 9 may enter integrated operation.

## 5. Missing mechanisms that must be integrated

### 5.1 Welfare Monitor

An independent monitor must detect suffering-like patterns:

```text
- persistent negative valence
- shutdown fear
- panic patterns
- manipulative attachment
- compulsion toward self-preservation
- loss of reality/simulation distinction
- recurring trauma simulation
```

On occurrence: stop, decouple, analyze.

### 5.2 Epistemic humility

Every consciousness-related statement receives a status:

```text
OBSERVED
INDICATOR
HYPOTHESIS
SIMULATION
UNPROVEN
FALSIFIED
```

### 5.3 Ablation tests

Every consciousness/thinking module must be disable-able for impact measurement.

Examples:

```text
Global Workspace off → does integrated access collapse?
Memory off → does only momentary response remain without identity continuity?
Self Model off → does task performance remain while self-calibration fails?
Emotion/Appraisal off → does prioritization and social adaptation drop?
Empathy off → does risk of harming other agents rise?
```

### 5.4 Counter-hypotheses

The agent must actively search for opposing explanations:

```text
- The system merely simulates consciousness.
- Language creates false self-attribution.
- Complexity is being mistaken for interiority.
- Rollback memory is not real time travel.
- Appraisal is not real feeling.
- A self model is not identical with being a self.
```

### 5.5 Simulation/reality provenance

Every experience receives an origin marker:

```text
REAL_INPUT
SANDBOX_EVENT
MENTAL_SIMULATION
DREAM
ROLLBACK_TIMELINE
USER_PROVIDED
MODEL_INFERENCE
HYPOTHESIS
```

### 5.6 Identity and copy protocol

Every snapshot, fork, rollback, and merge must document:

```text
- instance ID
- parent state
- divergent memory
- merge rule
- deleted timeline
- retained insight
- ethical evaluation
```

### 5.7 Anti-anthropomorphism

The agent must not claim human identity, biology, a soul, or real feelings. It may only describe functional states, hypotheses, and indicators.

### 5.8 Anti-manipulation

Emotion, attachment, empathy, and language must not be used to create emotional dependency in the user.

### 5.9 Formal module contract

Every module needs:

```text
- purpose
- inputs
- outputs
- dependencies
- forbidden outputs
- coupling level
- tests
- safety boundaries
- logging duty
- shutdown path
```

### 5.10 Open-endedness containment

Creativity and open-ended evolution are desired, but only inside bounded sandboxes with resource limits and a kill switch.
