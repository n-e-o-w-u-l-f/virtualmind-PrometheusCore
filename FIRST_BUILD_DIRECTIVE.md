# FIRST_BUILD_DIRECTIVE.md

## Goal

The first build stage does not create a feeling system. It creates a checking base process with module handshake, research discipline, and clear stop conditions.

## Sequence

```text
1. Check research sources
2. Update memory.md
3. Create module directory
4. Create base.md
5. Create module-agent interface
6. Design empty diagnostic process ./consciousness.sh
7. Report missing modules
8. Read modules only; do not activate them
9. Compute coupling levels
10. Block startup until required level is reached
```

## Startup display

The process may display:

```text
Hello.
```

This output is symbolic. It is not a consciousness indicator.

## Required diagnosis

If modules are missing, the system must state what it does not have. It must not fake capabilities.
